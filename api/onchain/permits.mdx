---
title: "List Permits"
api: "GET /api/onchain/permits"
---

Retrieves a list of recent permit requests (both approved and denied).

## Authentication

Requires a valid session token in the `Authorization` header:

```
Authorization: Bearer <session_token>
```

## Query Parameters

<ParamField query="days" type="integer" default="30">
  Number of days of history to retrieve (default: 30)
</ParamField>

<ParamField query="status" type="string">
  Filter by permit status: `approved` or `denied`. Omit to return all permits
</ParamField>

## Response

<ResponseField name="permits" type="array">
  Array of permit records
  
  <Expandable title="Permit object properties">
    <ResponseField name="id" type="integer">
      Database record ID
    </ResponseField>
    
    <ResponseField name="chain_id" type="integer">
      Blockchain chain ID
    </ResponseField>
    
    <ResponseField name="target" type="string">
      Target contract address
    </ResponseField>
    
    <ResponseField name="value" type="string">
      Transaction value as decimal string (uint256)
    </ResponseField>
    
    <ResponseField name="status" type="string">
      `"approved"` or `"denied"`
    </ResponseField>
    
    <ResponseField name="reason" type="string | null">
      Denial reason (only present for denied permits)
    </ResponseField>
    
    <ResponseField name="permit_hash" type="string | null">
      Keccak256 hash of the permit signature (only for approved permits)
    </ResponseField>
    
    <ResponseField name="cost_usd" type="number">
      Transaction value in USD
    </ResponseField>
    
    <ResponseField name="timestamp" type="integer">
      Unix timestamp when the permit was requested
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

<ResponseField name="error" type="string">
  Error message
</ResponseField>

### Status Codes

- `200` - Success
- `401` - Unauthorized (missing or invalid session token)
- `500` - Database error

## Examples

<CodeGroup>
```bash cURL
# Get all permits from last 7 days
curl "https://localhost:3978/api/onchain/permits?days=7" \
  -H "Authorization: Bearer fn_sess_abc123..."

# Get only denied permits
curl "https://localhost:3978/api/onchain/permits?status=denied" \
  -H "Authorization: Bearer fn_sess_abc123..."
```

```python Python
import requests

# Get all permits from last 7 days
response = requests.get(
    "https://localhost:3978/api/onchain/permits",
    headers={"Authorization": "Bearer fn_sess_abc123..."},
    params={"days": 7}
)
permits = response.json()["permits"]

# Get only approved permits
response = requests.get(
    "https://localhost:3978/api/onchain/permits",
    headers={"Authorization": "Bearer fn_sess_abc123..."},
    params={"status": "approved"}
)
approved = response.json()["permits"]
```

```javascript JavaScript
// Get all permits from last 7 days
const response = await fetch(
  'https://localhost:3978/api/onchain/permits?days=7',
  { headers: { 'Authorization': 'Bearer fn_sess_abc123...' } }
);
const { permits } = await response.json();

// Get only denied permits
const deniedResponse = await fetch(
  'https://localhost:3978/api/onchain/permits?status=denied',
  { headers: { 'Authorization': 'Bearer fn_sess_abc123...' } }
);
const { permits: denied } = await deniedResponse.json();
```
</CodeGroup>

## Use Cases

- Monitor recent onchain activity
- Debug why specific permits were denied
- Audit transaction history
- Track spending patterns by chain or contract
