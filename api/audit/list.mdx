## GET /api/audit

Retrieve audit log entries with pagination and filtering support. All API requests proxied through Fishnet are logged with cryptographic integrity guarantees.

### Authentication

Requires session authentication via Bearer token in the `Authorization` header.

```bash
Authorization: Bearer fn_sess_...
```

### Query Parameters

<ParamField query="from" type="integer" optional>
  Start timestamp in milliseconds (Unix epoch). Filters entries with `timestamp >= from`.
</ParamField>

<ParamField query="to" type="integer" optional>
  End timestamp in milliseconds (Unix epoch). Filters entries with `timestamp <= to`.
</ParamField>

<ParamField query="service" type="string" optional>
  Filter by service name (e.g., `openai`, `anthropic`, `binance`, `custom.github`).
</ParamField>

<ParamField query="decision" type="string" optional>
  Filter by policy decision. Values: `approved`, `denied`.
</ParamField>

<ParamField query="page" type="integer" optional default="1">
  Page number (1-indexed). Minimum value is 1.
</ParamField>

<ParamField query="page_size" type="integer" optional default="20">
  Number of entries per page. Valid range: 1-200.
</ParamField>

### Response

Returns a JSON object with paginated audit entries:

<ResponseField name="entries" type="array">
  Array of audit log entries, ordered by ID descending (newest first).
  
  <Expandable title="Audit Entry Schema">
    <ResponseField name="id" type="integer">
      Unique auto-incrementing entry ID.
    </ResponseField>
    
    <ResponseField name="timestamp" type="integer">
      Unix timestamp in milliseconds when the request was processed.
    </ResponseField>
    
    <ResponseField name="intent_type" type="string">
      Type of intent. Currently always `api_call`.
    </ResponseField>
    
    <ResponseField name="service" type="string">
      Service name (e.g., `openai`, `anthropic`, `binance`, `custom.{name}`).
    </ResponseField>
    
    <ResponseField name="action" type="string">
      HTTP method and path (e.g., `POST /v1/chat/completions`).
    </ResponseField>
    
    <ResponseField name="decision" type="string">
      Policy decision result: `approved` or `denied`.
    </ResponseField>
    
    <ResponseField name="reason" type="string | null">
      Human-readable explanation when request is denied. Null for approved requests.
    </ResponseField>
    
    <ResponseField name="cost_usd" type="number | null">
      Cost in USD for the request (LLM API calls, Binance orders). Null if not applicable.
    </ResponseField>
    
    <ResponseField name="policy_version_hash" type="string">
      Keccak256 hash of the policy configuration at the time of the request (hex-encoded, 64 chars).
    </ResponseField>
    
    <ResponseField name="intent_hash" type="string">
      Keccak256 hash of the request intent (method, service, action, query, body). Used for tamper detection.
    </ResponseField>
    
    <ResponseField name="permit_hash" type="string | null">
      Reserved for future on-chain permit integration. Currently null.
    </ResponseField>
    
    <ResponseField name="merkle_root" type="string">
      Merkle root hash at this point in the audit log. Enables cryptographic verification of log integrity.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of entries matching the filter criteria.
</ResponseField>

<ResponseField name="page" type="integer">
  Current page number.
</ResponseField>

<ResponseField name="pages" type="integer">
  Total number of pages available.
</ResponseField>

### Examples

<CodeGroup>

```bash curl
curl -X GET "http://localhost:3080/api/audit?page=1&page_size=10&service=openai&decision=approved" \
  -H "Authorization: Bearer fn_sess_YOUR_TOKEN"
```

```python Python
import requests

url = "http://localhost:3080/api/audit"
headers = {"Authorization": "Bearer fn_sess_YOUR_TOKEN"}
params = {
    "page": 1,
    "page_size": 10,
    "service": "openai",
    "decision": "approved"
}

response = requests.get(url, headers=headers, params=params)
data = response.json()

print(f"Total entries: {data['total']}")
for entry in data['entries']:
    print(f"{entry['id']}: {entry['service']} - {entry['action']} - {entry['decision']}")
```

```javascript JavaScript
const response = await fetch(
  "http://localhost:3080/api/audit?page=1&page_size=10&service=openai&decision=approved",
  {
    headers: {
      Authorization: "Bearer fn_sess_YOUR_TOKEN"
    }
  }
);

const data = await response.json();
console.log(`Total entries: ${data.total}`);
data.entries.forEach(entry => {
  console.log(`${entry.id}: ${entry.service} - ${entry.action} - ${entry.decision}`);
});
```

</CodeGroup>

### Error Responses

<ResponseField name="401 Unauthorized" type="object">
  ```json

  ```
  Session token is missing, invalid, or expired.
</ResponseField>

<ResponseField name="500 Internal Server Error" type="object">
  ```json
  {"error": "database error: ..."}
  ```
  Failed to query the audit database.
</ResponseField>

### Implementation Notes

- Entries are always returned in descending order by ID (newest first)
- The `page_size` parameter is clamped to the range [1, 200]
- The `page` parameter is automatically set to at least 1
- Timestamps are stored as 64-bit integers (milliseconds since Unix epoch)
- All hash fields are hex-encoded Keccak256 hashes (64 characters)
- The Merkle root enables cryptographic verification that the audit log has not been tampered with
